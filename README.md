# 演算法執行策略回測框架 (Algorithmic Execution Strategies Backtester)

這個專案是一個**「演算法交易執行策略」**的回測與模擬框架。

在機構交易（例如避險基金或大型資產管理公司）中，當投資組合經理（PM）決定要買入或賣出極大量的股票（例如 50 萬股的蘋果公司股票）時，他們不能直接將一筆市價單丟進市場，這會瞬間抽乾市場流動性（Liquidity），導致價格暴漲或暴跌，產生極大的損失。

這個平台就是在模擬並解決這個問題：**如何將一張「母單（Parent Order）」聰明地拆解成好幾張「子單（Child Orders）」，在一段時間內滴水穿石地將部位消化掉，以追求最低的交易摩擦成本。**

---

## 一、 專案核心模組與功能

1. **市場行情生合成器 (Mock Intraday Data)**
   - 使用**幾何布朗運動 (Geometric Brownian Motion)** 來模擬股價走勢，並刻意設定了美股常見的**「微笑曲線 / U 型交易量特徵 (U-Shaped Volume Profile)」**——也就是開盤與收盤時交易量大、中午時段交易量萎縮。
2. **撮合與回測引擎 (Matching Engine + Capacity Constraint)**
   - 真實世界的流動性是有限的。引擎內建了流動性限制功能，當演算法在某分鐘想買進 5000 股，但市場那一分鐘總量只有 2000 股時，引擎會予以限制，無法超額成交。
3. **市場衝擊模型 (Market Impact Model)**
   - 專案寫入了一個非線性的**「暫時性市場衝擊成本函數」**。其概念是：訂單量佔每日平均交易量 (Average Daily Volume, ADV) 的比例越高，成交的價格就會越差（被懲罰）。並透過 `statsmodels` 對數迴歸法 (Log-Log OLS) 精準配適出 \(\alpha\) 與 \(\beta\) 參數。
4. **TWAP 策略 (Time-Weighted Average Price)**
   - 「時間加權平均價格策略」。它的作法非常粗暴，即單純將時間等分（例如每 5 分鐘固定丟出一筆等量的單），完全不管當下市場是熱絡還是冷清。
5. **VWAP 策略 (Volume-Weighted Average Price)**
   - 「成交量加權平均價格策略」。它會根據預測的市場交易量輪廓 (Volume Profile) 來改變拆單大小。市場量大（開收盤）它就買得多，市場量小它就買得少，以此「隱藏」在市場正常的成交量之中。

## 二、 模擬結果與金融語言解讀

在 Streamlit 儀表板上，我們透過隨機生成上百筆訂單產生的圖表，可以用以下金融專業觀點來解讀：

### 1. Implementation Shortfall (執行落差 / 滑價成本)
這也就是圖表中的 **IS (bps)**，單位是基點（1 Basis Point = 0.01%）。
在金融機構中，我們通常以**「決策到達價 (Arrival Price)」**——也就是當你決定要下單那一瞬間的市場中價——作為基準。
- **解讀：** 最終所有子單的「平均成交價 (Average Execution Price)」與「到達價」之間的差距就是 IS。如果圖表上顯示的是 `-45 bps`，代表策略為了把這筆巨量訂單吞完，平均成交價比理想價格差了萬分之四十五，這就是**因為下大單而付出的隱性交易成本**。

### 2. TWAP vs VWAP 常態分配差異 (KDE Distribution)
在第一張鐘形曲線圖中，VWAP 的分佈很集中，而 TWAP 的分佈很寬（變異數大）。
- **解讀（Execution Risk / 執行風險）：** 
TWAP 因為強迫在固定時間硬吃固定數量的股票，若剛好在那段時間市場嚴重缺乏流動性（Illiquid），這筆訂單就會對盤口產生極嚴重的**「逆向選擇 (Adverse Selection)」**，把當下價格掃空，付出極高的滑價。反觀 VWAP 完美跟隨了流動性，有效地降低了「執行跟蹤誤差 (Tracking Error)」，讓成本分佈極度可控與穩定。

### 3. 成本曲線與訂單大小的非線性關係 (Market Impact vs Order Size)
在第二張散佈圖中，隨著訂單大小 (Size) 往右增加，VWAP 上升的斜率較為平緩，而 TWAP 的成本則呈現爆炸性噴發。
- **解讀（Market Capacity / 市場承載力）：**
這完美展示了量化交易中的**「平方根衝擊法則 (Square-Root Impact Law)」**。小單不會影響市場，但當大單超過了市場承載力時，流動性枯竭會導致遭到市場無情的剝削。VWAP 因為懂得把大單隱藏在開盤/收盤的高流動池中，其市場衝擊（Market Impact）上升的幅度遠比無腦買入的 TWAP 來得低。

---

## 結論

> 「這套框架證明了，當單支股票交易量突破 ADV 的一定比例後，純時間切片的 TWAP 會因為承受過高的『流動性風險』而導致執行落差 (IS) 面臨失控的尾部風險。我們切換至具備成交量預期 (Volume Profile) 的 VWAP 策略，藉此成功最小化了『暫時性市場衝擊成本』，為基金保留了更多的預期阿爾法 (Alpha)。」
